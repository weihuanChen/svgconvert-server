# SVG 转换工具产品需求文档 (PRD)

项目名称： SVG Utility Tool (暂定)

域名建议： [YourDomain].net

版本： 1.0

日期： 2025年10月

## 1. 项目概述 (Project Overview)

本项目旨在创建一个高效、用户友好的在线工具，专注于 SVG（可缩放矢量图形）与其他主流图片及文档格式（如 JPG, PNG, PDF）之间的双向转换。核心价值是提供**高质量、快速、多语言支持**的转换服务。

### 1.1 项目目标

1. 实现 SVG 与 JPG/PNG/PDF 之间的**双向**、**无损或高保真**转换。
2. 将**日语 (ja)** 作为主要目标语言，实现完善的国际化支持。
3. 确保跨设备（桌面、移动端）的卓越用户体验。

### 1.2 目标用户

- **网页设计师和开发者：** 需要将矢量图快速转换为位图，或将位图转换为矢量图进行优化。
- **内容创建者：** 需要将 SVG 图标或插图转换为网站、报告或印刷品所需的格式。
- **办公用户：** 需要将图形转换为易于分享的 PDF 或标准图片格式。

## 2. 核心功能 (Core Features)

### 2.1 转换矩阵 (Conversion Matrix)

项目必须支持以下核心双向转换功能：

| **转换类型**        | **输入格式 (Input)** | **输出格式 (Output)**  |
| ------------------- | -------------------- | ---------------------- |
| **SVG 转位图/文档** | SVG                  | PNG, JPG, PDF          |
| **位图/文档转 SVG** | PNG, JPG, PDF        | SVG (需进行矢量化处理) |

### 2.2 转换设置与优化

用户在转换前必须能够配置以下参数：

#### SVG -> PNG/JPG 转换设置 (矢量转位图)

- **尺寸控制：** 用户可输入自定义的宽度（Width）和高度（Height），或选择保持原始宽高比（Aspect Ratio）。
- **背景色：** 可选择透明背景（仅限 PNG）或自定义背景色（如白色、黑色或十六进制代码）。
- **JPG 质量：** 可调节滑块（1% - 100%），默认为 85%。
- **PNG 优化：** 选项用于无损压缩 PNG 文件大小。

#### PNG/JPG -> SVG 转换设置 (位图转矢量)

- **矢量化算法：** 允许用户选择不同的矢量化模式（例如：边缘检测、色彩追踪、简化路径）。
- **色彩数量：** 控制生成 SVG 中使用的最大颜色数量。
- **平滑度：** 调节路径平滑度以平衡精度和文件大小。

### 2.3 批量处理 (Batch Processing)

- **多文件上传：** 允许用户一次性上传最多 5 个文件进行批量处理。
- **统一设置：** 转换设置（如尺寸、质量）应可应用于所有批量文件，也可为每个文件单独配置。

## 3. 用户界面与流程 (UI/UX Flow)

### 3.1 极简设计原则

- 设计必须**简洁、清晰、色彩柔和**，以提供专业和快速的工具感。
- 采用 **移动优先（Mobile-First）** 响应式设计，确保在手机上也能轻松上传和下载。

### 3.2 转换工作流 (Workflow)

1. **上传区域 (Drop Zone)：** 页面核心是一个醒目的拖放区域或上传按钮，提示用户上传文件。
   - **i18n Tip:** 核心提示语应为日文的 **ファイルアップロード** (File Upload) 或 **ここにファイルをドラッグ＆ドロップ** (Drag & Drop files here)。
2. **文件列表与选择：** 文件上传后，显示文件缩略图/名称，并让用户选择**目标格式**（例如：SVG -> PNG）。
3. **配置设置：** 显示上文提到的所有转换参数（尺寸、质量、背景等）。
4. **开始转换：** 一个清晰的、状态可变的按钮（例如：**変換開始** -> 正在转换...）。
5. **下载结果：** 转换完成后，提供单个文件下载链接或批量打包（Zip）下载。

## 4. 国际化 (i18n) 重点要求

国际化是本项目的**核心要求**。**日语 (ja)** 必须作为首要支持语言，并保持最高质量。

### 4.1 语言范围

| **语言名称** | **语言代码** | **状态**     | **备注**                             |
| ------------ | ------------ | ------------ | ------------------------------------ |
| **日语**     | `ja`         | **主要支持** | 所有 UI 元素必须经过专业的日文翻译。 |
| **中文**     | `zh`         | 初始支持     | 简体中文。                           |
| **英语**     | `en`         | 初始支持     | 默认备选语言。                       |

### 4.2 技术实现

- **资源文件：** 采用 JSON 格式存储所有 UI 文本和消息，按语言代码 (`ja.json`, `zh.json`, `en.json`) 分类。
- **运行时检测：** 优先使用浏览器或操作系统设置来确定用户的首选语言。
- **语言切换器：** 在页面的页眉（Header）或页脚（Footer）提供醒目的语言切换下拉菜单或按钮。
  - **注意：** 在日文模式下，语言切换器应显示为 **言語** 或 **Language**，而不是 "Japanese"。

### 4.3 日语翻译示例（必须实现）

| **英文原词**     | **中文** | **日文 (ja)**              | **用途**         |
| ---------------- | -------- | -------------------------- | ---------------- |
| Upload File      | 上传文件 | **ファイルをアップロード** | 上传按钮或提示   |
| Start Conversion | 开始转换 | **変換を開始**             | 转换触发按钮     |
| Download         | 下载     | **ダウンロード**           | 结果下载按钮     |
| Image Quality    | 图片质量 | **画質**                   | JPG 质量设置标签 |
| Transparency     | 透明度   | **透明度**                 | PNG 背景设置选项 |
| Settings         | 设置     | **設定**                   | 配置面板标题     |
| Batch Process    | 批量处理 | **一括処理**               | 模式切换或提示   |

## 5. 技术要求 (Technical Requirements)

### 5.1 架构与技术栈

- **前端：** 推荐使用 React/Vue/Angular 等现代前端框架，以实现组件化和高性能的 i18n 集成。
- **后端：** 需使用强大的后端语言（如 Python/Node.js）来处理 CPU 密集型的文件转换任务。
- **API 设计：** 所有转换任务必须通过异步 API 进行管理，确保用户界面不会因长时间转换而卡死。

### 5.2 性能与安全

- **处理速度：** 90% 的转换任务应在 10 秒内完成。
- **数据安全：** 上传的文件和转换结果在下载完成后（例如 30 分钟）必须从服务器上**自动清除**，以保护用户隐私。
- **文件限制：** 初始阶段，单个文件大小限制为 20MB。

## 6. 未来功能规划 (Future Roadmap)

1. **WebP 支持：** 增加对现代 WebP 格式的转换支持。
2. **AI 增强：** 集成基本的 AI 图像处理功能，如位图 SVG 转换后的**路径优化**。
3. **URL 转换：** 允许用户输入图片 URL，直接进行抓取和转换。
4. **API 服务：** 提供付费 API 接口，允许开发者在其应用中调用转换功能。
